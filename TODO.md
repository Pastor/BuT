## 1. Система типов

Необходимо скорректировать систему типов следующим образом: существуют только два базовых типа bit и float, остальные типы выводятся из них.
float является математическим типом и при генерации кода преобразуется соответственно в float - C, FLOAT - ST, в asm используются 
функции по работе с операндами с плавающей точкой. Из bit выводятся все остальные типы, как то u8 -> [8: bit], u16 -> [16: bit] и т.д.
соответственно при генерации кода они сводятся к типам генерируемого языка u8 -> unsigned char, u16 -> unsigned short и т.д. 
Типы, которые превышают размер базовых типов генерируемого языка приводятся к массивам с выравниванием по ближайшей границы простого типа.

## 2. Доступ до конкретного бита

Необходимо добавить операцию получения/изменения конкретного бита переменной не математического типа. Операция точка `.`.
Если передается имя переменной и после нее идет точка с числом - `.34` то надо считать что идет обращение к 35 
(начиная с 0, младшего адреса) биту переменной. Система типов должна делать проверку на уровне семантического анализа, что 
при обращении к биту нет выхода за границы типа. При кодогенерации используются языковые конструкции доступа к биту данных.

Примеры:
```
let a: [8: bit] = 0;

a.5 = true; // Выставление 6 бита в true
```

```
let b: [128: u8];

b.250 = false; // Обращение к 251 биту области памяти
```

```
let c: [b: bit]

c.64 = true; // Ошибка, выход за границы типа
```
